Prerequisites & Installation

▼

iact3 is developed in Python. Make sure you have Python 3.7+ installed.

Install via pip (recommended)

pip install alibabacloud-ros-iact3

Install from Source

git clone https://github.com/aliyun/alibabacloud-ros-tool-iact3.git
cd alibabacloud-ros-tool-iact3
python setup.py install

Configure Credentials

▼

iact3 supports three ways to configure Alibaba Cloud credentials:

Web UI (Recommended)

Go to Settings, enter your Access Key ID and Secret, then click Save Settings.

Alibaba Cloud CLI

If you have the CLI tool installed, iact3 can use its config file directly.

{
  "current": "default",
  "profiles": [
    {
      "name": "default",
      "mode": "AK",
      "access_key_id": "<ALIBABA_CLOUD_ACCESS_KEY_ID>",
      "access_key_secret": "<ALIBABA_CLOUD_ACCESS_KEY_SECRET>"
    }
  ]
}

Alibaba Cloud SDK Credentials

Uses the SDK credentials file.

[default]
enable=true
type=access_key
access_key_id=<ALIBABA_CLOUD_ACCESS_KEY_ID>
access_key_secret=<ALIBABA_CLOUD_ACCESS_KEY_SECRET>

[client1]
enable=true
type=ecs_ram_role
role_name=<ecs_ram_role_name>

[client2]
enable=true
type=ram_role_arn
access_key_id=<ALIBABA_CLOUD_ACCESS_KEY_ID>
access_key_secret=<ALIBABA_CLOUD_ACCESS_KEY_SECRET>
role_arn=<ram_role_arn>
role_session_name=<role_session_name>

Environment Variables

Set the following environment variables before starting iact3:

export ALIBABA_CLOUD_ACCESS_KEY_ID=<your-access-key-id>
export ALIBABA_CLOUD_ACCESS_KEY_SECRET=<your-access-key-secret>

Configuration File Guide (.iact3.yml)

▼

The config file .iact3.yml defines how iact3 tests your template. It has three sections:

Minimal Example

project:
  name: my-first-test
  template_config:
    template_url: 'https://xxx/create-ecs.yaml'
  regions:
    - cn-hangzhou
    - cn-beijing
tests:
  test-name-1:
    parameters:
      ZoneId: $[iact3-auto]
      InstanceType: ecs.g6e.large
  test-name-2:
    parameters:
      ZoneId: $[iact3-auto]
      InstanceType: ecs.c6.large

Full Example

general:                    # Global defaults (optional)
  auth:                     # Aliyun CLI credentials
    name: default
    location: ~/.aliyun/config.json
  oss_config:               # OSS bucket for uploading results
    bucket_name: my-bucket
    bucket_region: cn-hangzhou
  parameters:               # Shared parameters for all tests
    VpcId: $[iact3-auto]    # Auto-generate VPC for testing
    VSwitchId: $[iact3-auto]

project:                    # Project settings
  name: my-project
  regions:
    - cn-hangzhou
    - cn-beijing
  template_config:          # Specify template location
    template_location: templates/ecs.yaml
  parameters:
    InstanceChargeType: PostPaid
    Password: $[iact3-auto] # Auto-generate password
  tags:
    environment: test

tests:                      # Test cases (required)
  default:                  # Test name 1
    parameters:
      InstanceType: ecs.g6.large
    regions:
      - cn-hangzhou
    hooks:                  # Lifecycle hooks
      pre-create:
        execute_time: PreCreate   # PreCreate | PostCreate | PreDelete | PostDelete
        execute_command: [echo, hello]
  large-instance:           # Test name 2
    parameters:
      InstanceType: ecs.g6.2xlarge

Key Fields Explained

FieldRequiredDescription
testsYesTest cases. Each test defines parameters, regions, hooks, etc.
project.nameNoProject name, used in output file naming.
project.regionsNoDefault regions for testing. Can be overridden per test.
project.template_configNoSpecify template via template_location, template_url, or template_id.
project.parametersNoParameter key-values passed to the template.
general.authNoAlibaba Cloud CLI credentials config (name & location of config.json).
general.oss_configNoOSS bucket config for uploading test results.
general.parametersNoGlobal parameters. Unlike other fields, more general values take priority for parameters.
$[iact3-auto]-Special value: iact3 auto-creates base resources (VPC, VSwitch, etc.) for testing.

Notes on Config File Usage in Web UI

In the Web UI, you can paste config content directly into the Config File editor.

Save content as a Project to store template and config together.

If the Config File editor is empty when you start a test, a minimal config is auto-generated.

CLI Commands Reference

▼

Besides the web UI, iact3 can be used via CLI.

CommandDescription
iact3 test runRun the full test suite — create stacks, test, and auto-cleanup.
iact3 validateValidate template syntax and structure.
iact3 previewPreview what resources will be created.
iact3 costEstimate resource pricing (inquiry).
iact3 policyQuery the RAM policy required for deployment.
iact3 listList all test runs and their statuses.
iact3 deleteDelete a specific test run.
iact3 -hShow help info for all available commands.

Frequently Asked Questions

▼
Q: What template formats are supported?

A: ROS templates (.json, .yaml, .yml) and Terraform templates (.tf).

Q: What does $[iact3-auto] mean?

A: It tells iact3 to automatically create base resources (VPC, VSwitch, Security Group, etc.) for testing.

Q: Where are the test results stored?

A: In the iact3_outputs/ directory.

Q: Do Validate/Policy need regions?

A: No. Only Preview and Cost require regions to be specified.

Q: Can I use environment variables for credentials?

A: Yes. Set ALIBABA_CLOUD_ACCESS_KEY_ID and ALIBABA_CLOUD_ACCESS_KEY_SECRET before starting the server.

Q: How do I start the web server?

A: Run: iact3 server start --host 0.0.0.0 --port 8088

Q: I submitted a test but no stacks were created — why?

A: Make sure at least one region is selected. Also ensure you have valid credentials saved in Settings. Check Dashboard and click the run to see detailed error logs.
